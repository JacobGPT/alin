Can you explain how WebSockets differ from Server-Sent Events?
Claude Sonnet 4.5
Great question. The key difference is directionality. WebSockets create a full-duplex connection ‚Äî both client and server can send messages at any time. SSE is one-directional: the server pushes events to the client, but the client can only send data via regular HTTP requests.

For ALIN's streaming responses, SSE is actually the better choice because we only need server‚Üíclient streaming for token delivery.
